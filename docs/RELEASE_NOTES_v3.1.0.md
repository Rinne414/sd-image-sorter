## v3.1.0 版本说明

这版不要从工程角度看，直接从用户角度看：

- Reader 现在不只是看图了，可以直接改 metadata 再另存新图。
- 扫描大图库时，会更早把图库先显示出来，不用傻等全部跑完。
- 后台还在继续导入时，画面会明确告诉你“还在继续”，而不是像卡死。
- 覆盖保存更顺，先确认再覆盖，不会先报错一次再问你。
- 在 WSL 跑后端时，旧的 Windows 路径图库缩略图也更稳定了。

---

## 你会直接感觉到的变化

### 1. Reader 终于能直接改 metadata

现在在 Reader 里，用户可以直接修改这些内容：

- Prompt
- Negative Prompt
- Seed
- Sampler
- Steps
- CFG
- Size
- Model / Checkpoint
- LoRA

改完后可以直接另存成一张新图片，不用再手动复制文本去别的工具处理。

另外，保存时也能直接选输出格式：

- PNG
- WebP
- JPG

---

### 2. 覆盖保存不再先失败一次

以前同路径保存的体验很蠢：

- 先撞一次冲突
- 再弹确认

现在改成正常流程：

- 先问你要不要覆盖
- 你确认后才真的执行

这对连续修图、连续改 metadata 的用户会顺很多。

---

### 3. 扫描大图库时，更早看到结果

这版最重要的体验变化之一，是扫描时不再尽量把用户卡在“等全部准备完”。

现在的方向是：

- 先尽快把已发现的图片导入图库
- 让用户先开始浏览
- 剩下的图片和 metadata 再继续在后台补

对用户来说，区别不是“某个技术细节变了”，而是：

- 更早看到图库内容
- 更早知道软件没有卡死
- 更早开始筛图，而不是一直等

---

### 4. 扫描进度更像正常软件

大图库用户最怕两件事：

- 不知道软件是不是死了
- 不知道现在是在扫，还是已经卡住

这版把进度表达改得更接近真实使用感：

- 图库可先用时，会明确告诉你
- 后台还在继续导入时，会持续显示状态
- 不再硬装一个看起来很“完整”但其实不真实的进度感

简单说，就是少一点假进度，多一点真实反馈。

---

### 5. WSL 用户的 Windows 路径图库更稳

如果你是这种用法：

- 图片放在 Windows 盘
- 后端跑在 WSL

这版补了旧路径兼容，像 `L:\...` 这种老路径会更正常地映射到 WSL 挂载路径。

用户能直接感受到的是：

- 缩略图更不容易 404
- 打开图库时更不容易出现“明明图在，软件却说找不到”的感觉

---

### 6. JPG / WebP 的提示终于说人话

这版不会再假装所有格式都和 PNG 一样适合保 metadata。

现在会更明确告诉用户：

- PNG 仍然是最稳的保存格式
- JPG / WebP 可以用
- 但 metadata 保留能力没有 PNG 那么可靠

这不是炫技，是减少用户保存后才发现信息不见的挫败感。

---

## 这版适合谁

v3.1.0 特别适合这几类用户：

- 会改 prompt / seed / model 信息的人
- 需要把 Reader 当整理工具，而不是单纯预览器的人
- 图很多，扫描时最怕等太久的人
- 在 WSL 跑后端、图片放 Windows 盘的人

---

## 一句话总结

v3.1.0 的重点不是“多了一个技术功能”，而是把几条关键体验做得更像真正能长期用的软件：

- Reader 更能用
- 保存更顺
- 扫描更早给结果
- 后台状态更清楚
- 大图库时没那么像死机

---

## 验证情况

这版不是只改文案：

- Reader 关键保存流程已经做过真实浏览器验证
- 扫描与 metadata 相关回归也已重新跑过

如果你要给普通用户看，这版可以直接用上面这份说明，不需要再讲底层实现。
